# Architecture Document Content

1.交易规则

交易记账：交易双方确认交易后，将交易信息，包括：付款方ID、收款方ID、交易时间、交易金额、交易详情（具体的商品/服务）存档至各自区块链。

交易汇报：交易双方将交易的时间、金额、交易对方所在小组ID及交易哈希值汇报给本小组核心节点。

交易确认：付款方小组核心节点确认付款方有足够余额可支付；收款方所在小组核心节点向付款方所在小组核心节点确认交易是否有效。

有效条件：交易双方所在小组核心节点收到了一致的交易信息且付款方有足够余额可支付。

节点余额：核心节点掌握本小组其它节点收支情况，因而掌握节点余额。收支汇总：核心节点汇总本小组（含子小组）收支情况并逐级上报，确保社区整体收支相符（即各小组声明的收入之和与支出之和相等）。

汇总确认：顶级小组将确认后的汇总收支情况写入社区区块。
